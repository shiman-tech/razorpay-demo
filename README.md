# 💳 Razorpay Payment Gateway Integration using Flutter + FastAPI

A complete end-to-end payment gateway integration built using **Flutter**, **FastAPI**, and **Razorpay**. This project demonstrates how a mobile application can securely create orders, initiate payments, verify signatures, and update the user interface after successful payment verification.

---

## 📌 Project Overview

This project was built to understand the complete lifecycle of an online payment:

1. Create an order from the backend.
2. Open Razorpay Checkout in Flutter.
3. Process payment using Razorpay.
4. Receive payment callbacks.
5. Verify payment signatures on the backend.
6. Update the UI based on verification results.
7. Follow clean architecture principles by separating concerns into services, models, and screens.

The goal of this project is educational: to learn how real-world payment gateways are integrated into modern applications.

# 🔄 Payment Flow

## Step 1: Create Order

Flutter requests the backend to create a Razorpay order.

```text
Flutter
   ↓
POST /create-order
   ↓
FastAPI
   ↓
Razorpay
```

The backend returns:

```json
{
  "id": "order_xxxxxxxxx",
  "amount": 50000,
  "status": "created"
}
```

---

## Step 2: Open Checkout

Flutter uses the received order ID to launch Razorpay Checkout.

```text
Order Created
      ↓
Razorpay Checkout Opens
```

---

## Step 3: User Completes Payment

Razorpay returns:

```text
payment_id
order_id
signature
```

through the success callback.

---

## Step 4: Verify Payment

Flutter sends:

```json
{
  "razorpay_order_id": "...",
  "razorpay_payment_id": "...",
  "razorpay_signature": "..."
}
```

to the backend.

---

## Step 5: Signature Verification

Backend verifies the payment:

```python
client.utility.verify_payment_signature(...)
```

If verification succeeds:

```json
{
  "verified": true
}
```

Otherwise:

```json
{
  "verified": false
}
```

# 🔐 Security Design

## Public Key

Used inside Flutter.

```text
RZP_KEY_ID
```

Safe to expose.

---

## Secret Key

Used only on backend.

```text
RZP_SECRET
```

Never exposed to Flutter.

---

## Why Verification Is Required

Anything sent from the client can be modified.

Without signature verification, a malicious user could fake:

```text
payment_id
order_id
payment success
```

Verification ensures the payment was genuinely generated by Razorpay.

# 🧠 Architecture Decisions

## Why Separate Services?

### PaymentService

Responsible for:

```text
Backend Communication
```

Examples:

```text
Create Order
Verify Payment
```

---

### RazorpayService

Responsible for:

```text
Razorpay SDK Interaction
```

Examples:

```text
Checkout
Success Events
Failure Events
```

---

### HomePage

Responsible for:

```text
UI Rendering
State Updates
User Interaction
```

---

This separation follows the principle of:

```text
Single Responsibility
```

where each component has one clear purpose.
